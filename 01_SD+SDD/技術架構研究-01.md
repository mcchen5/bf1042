## 專案概述

### 功能需求
- RESTful API 提供菜單、訂單管理
- 使用者認證（Google 第三方登入）
- AI 自然語言訂餐（口語轉換為菜色）
- PDF 合約上傳、驗證（簽名、印章檢測）
- OCR 發票辨識
- WebSocket 即時通知

### 技術約束
- 使用 Bun 運行時
- TypeScript 全端開發
- 繁體中文優化
- 成本可控（個人/小型團隊）

## 後端框架

### 選型：Elysia

#### 為何不選原生 Bun.serve？

| 特性 | Bun 原生 | Elysia |
|------|----------|--------|
| 類型安全 | ❌ 手動定義 | ✅ 端對端推導 |
| Schema 驗證 | ❌ 手動檢查 | ✅ Zod/TypeBox 內建 |
| 自動 API 文件 | ❌ 無 | ✅ Swagger/OpenAPI |
| 中間件生態 | ❌ 自己寫 | ✅ JWT、CORS、壓縮等 |
| 錯誤處理 | ❌ try-catch 分散 | ✅ 統一 onError hook |

#### Elysia 核心優勢

```typescript
// 類型安全自動推導
new Elysia()
  .post("/api/orders", 
    ({ body }) => {  // body 自動推導為 OrderType
      return { data: createOrder(body) }
    },
    { body: t.Object({ name: t.String(), price: t.Number() }) }
  )
```

#### Eden Treaty（類型安全 HTTP 客戶端）

```typescript
// 前端自動繼承後端類型
const api = treaty<App>('http://localhost:3000')
const { data } = await api.api.menu.get()  // 自動推導 MenuItem[]
```

## 資料庫與 ORM

### 選型：PostgreSQL + Drizzle ORM

#### 為何選 Drizzle？

| ORM | 與 Elysia 整合 | 特點 |
|-----|---------------|------|
| **Drizzle** | ⭐⭐⭐⭐⭐ | SQL-like 語法、零運行時、完美類型支援 |
| Prisma | ⭐⭐⭐⭐ | 需 generate、運行時較重 |
| TypeORM | ⭐⭐⭐ | 裝飾器語法、類型推導較弱 |

#### drizzle-typebox 自動生成驗證

```typescript
import { createInsertSchema } from 'drizzle-typebox'

export const menuItems = pgTable('menu_items', {
  id: serial('id').primaryKey(),
  name: varchar('name').notNull(),
  price: integer('price').notNull()
})

// 自動生成 Elysia 驗證 Schema
export const insertMenuSchema = createInsertSchema(menuItems)
```

#### 向量搜索（pgvector）

```typescript
// AI 訂餐的語義匹配
const similarItems = await db
  .select({
    name: menuItems.name,
    similarity: sql`1 - (${cosineDistance(menuItems.embedding, queryEmbedding)})`
  })
  .from(menuItems)
  .orderBy(cosineDistance(menuItems.embedding, queryEmbedding))
  .limit(3)
```

## 認證與授權

### 選型：Lucia + Arctic（Google OAuth）

#### 方案比較

| 方案 | 適用場景 | 特點 |
|------|---------|------|
| **Lucia + Arctic** | 純 API 架構 | 輕量、與 Drizzle 整合、完全可控 |
| Better Auth | Next.js 全端 | 基於 Cookie，與 Elysia 整合較複雜 |
| Clerk | 快速上線 | 商業方案、免費額度 10,000 MAU |

#### 為何不選 Better Auth？

Better Auth 設計給全端框架（Next.js），在 Elysia 純 API 架構中：
- Cookie 處理需額外 CORS 配置
- Session 表結構可能與現有 schema 衝突
- 預設接管 `/api/auth/*` 路由

#### Lucia 實作範例

```typescript
import { Lucia } from 'lucia'
import { DrizzlePostgreSQLAdapter } from '@lucia-auth/adapter-drizzle'

const adapter = new DrizzlePostgreSQLAdapter(db, sessions, users)

export const lucia = new Lucia(adapter, {
  sessionCookie: {
    attributes: { secure: process.env.NODE_ENV === 'production' }
  },
  getUserAttributes: (attributes) => ({
    email: attributes.email,
    name: attributes.name
  })
})
```

## OCR 服務

### 選型：Google Cloud Vision API

#### 方案比較

| 方案 | 適用場景 | 優缺點 |
|------|---------|--------|
| **Google Vision** | 繁體中文、發票 | 準確率最高、免費 1000 次/月 |
| Tesseract.js | 離線需求 | 免費但中文準確率較低 |

#### 費用
- 每月 1000 次免費額度
- 超出後 $1.5/1000 次

#### 實作範例

```typescript
class OCRService {
  async detectText(base64Image: string) {
    const response = await fetch(
      `https://vision.googleapis.com/v1/images:annotate?key=${this.apiKey}`,
      {
        method: 'POST',
        headers: { 'Content-Type': 'application/json' },
        body: JSON.stringify({
          requests: [{
            image: { content: base64Image },
            features: [{ type: 'TEXT_DETECTION' }]
          }]
        })
      }
    )
    // 解析回應...
  }
  
  // 發票專用：文件模式
  async detectReceipt(base64Image: string) {
    const response = await fetch(
      `https://vision.googleapis.com/v1/images:annotate?key=${this.apiKey}`,
      {
        method: 'POST',
        body: JSON.stringify({
          requests: [{
            image: { content: base64Image },
            features: [
              { type: 'DOCUMENT_TEXT_DETECTION' },  // 更準確
              { type: 'OBJECT_LOCALIZATION' }
            ]
          }]
        })
      }
    )
    
    // 解析統編、金額
    const taxId = text.match(/\b\d{8}\b/)?.[0]
    const amount = text.match(/(?:總計|Total).*?[\$NT]?\s*(\d+)/i)?.[1]
    return { taxId, amount: amount ? parseInt(amount) : null }
  }
}
```

## PDF 處理與儲存

### 檔案儲存：Dropbox（既有 1TB 訂閱）

#### 為何使用 Dropbox？

| 面向 | 評估 |
|------|------|
| **成本** | ⭐⭐⭐⭐⭐ 已付費，零額外成本 |
| **容量** | ⭐⭐⭐⭐⭐ 1TB 非常充足 |
| **可靠性** | ⭐⭐⭐⭐☆ Dropbox 服務穩定 |
| **限制** | API 速率限制（每秒 ~100 次） |

#### API 速率限制處理

```typescript
async function withDropboxRetry<T>(operation: () => Promise<T>, maxRetries = 3): Promise<T> {
  for (let i = 0; i < maxRetries; i++) {
    try {
      return await operation()
    } catch (error: any) {
      if (error.status === 429) {
        const retryAfter = error.error?.retry_after || Math.pow(2, i)
        await new Promise(r => setTimeout(r, retryAfter * 1000))
        continue
      }
      throw error
    }
  }
  throw new Error('Max retries exceeded')
}
```

### PDF 驗證流程

```
PDF 上傳
    ↓
文字提取 (pdf-parse) → 關鍵字驗證（姓名、電話）
    ↓
轉換圖片 (pdf-to-img)
    ↓
簽名檢測 (Google Vision 手寫識別)
印章檢測 (顏色/形狀分析)
    ↓
驗證結果寫入資料庫
```

## AI 自然語言訂餐

### 選型：Kimi (Moonshot)

#### 方案比較

| 模型 | 中文理解 | 上下文 | 價格 |
|------|---------|--------|------|
| **Kimi** | ⭐ 極佳 | 200K | 較低 |
| GPT-4o-mini | 良好 | 128K | $0.15/M |
| Gemini Flash | 良好 | 1M | 免費額度 |

#### 為何選 Kimi？

1. **中文口語理解最佳**：專為中文優化，懂「那個脆脆的、很台灣的」
2. **200K 上下文**：適合多輪對話記憶
3. **兼容 OpenAI SDK**：程式碼幾乎不用改

#### 實作架構

```typescript
// 1. 建立菜品向量資料庫
async function initializeEmbeddings(menuItems: MenuItem[]) {
  for (const item of menuItems) {
    const richDescription = `
      品名：${item.name}
      別名：${item.aliases?.join('、')}
      口感：${item.tags?.join('、')}
      成分：${item.ingredients?.join('、')}
    `
    const { embedding } = await embed({
      model: openai.embedding('text-embedding-3-small'),
      value: richDescription
    })
    // 存入資料庫...
  }
}

// 2. 智能匹配引擎
class SmartOrderMatcher {
  async match(userInput: string) {
    // AI 理解意圖
    const intent = await this.aiParser.parse(userInput)
    
    // 語義搜索 Top 3 候選
    const candidates = await this.embeddingService.semanticSearch(intent.items[0].name, 3)
    
    // LLM 最終判斷
    const bestMatch = await this.selectBestMatch(intent, candidates)
    
    return {
      matchedItem: bestMatch.item,
      confidence: bestMatch.confidence,
      needsConfirmation: bestMatch.confidence < 0.85
    }
  }
}
```

## 雲服務選型

### PostgreSQL：Neon（Serverless Postgres）

#### 為何選 Neon？

| 服務 | 免費額度 | 特點 |
|------|---------|------|
| **Neon** | 0.5GB / 190hr | Serverless、自動擴展、Bun 兼容 |
| Supabase | 500MB | 內建 Auth、儀表板完整 |

**Neon 優勢**：
- `neon-http` 驅動專為邊緣優化，Bun 環境完美支援
- Branching：每個 PR 自動創建 DB 分支測試
- 閒置時幾乎免費

### 應用部署：Fly.io

#### 為何選 Fly.io？

| 平台 | Bun 支援 | 延遲（台灣） | 成本 |
|------|---------|-------------|------|
| **Fly.io** | ⭐⭐⭐⭐⭐ | < 50ms（新加坡） | $2-5/月 |
| Railway | ⭐⭐⭐⭐ | 中高 | $5+/月 |
| Render | ⭐⭐⭐ | 較慢 | $7+/月 |
| Hostinger VPS | ⭐⭐⭐⭐⭐ | 取決機房 | $4-8/月 |

**Fly.io 優勢**：
- 官方 `oven/bun` 鏡像，原生支援
- 新加坡/東京節點，台灣延遲極低
- Auto-suspend：閒置自動休眠省錢

#### fly.toml 配置

```toml
app = "breakfast-api"
primary_region = "sin"  # 新加坡

[http_service]
  internal_port = 3000
  force_https = true
  auto_stop_machines = true
  auto_start_machines = true
  min_machines_running = 0  # 閒置停機省錢

[[vm]]
  size = "shared-cpu-1x"
  memory = "512mb"
```

## 技術風險與因應

| 風險 | 影響 | 因應措施 |
|------|------|---------|
| Kimi API 不穩定 | AI 訂餐功能失效 | 實作 fallback 到 Gemini |
| Dropbox API 限流 | 檔案上傳失敗 | 請求佇列 + 重試機制 |
| Fly.io 冷啟動 | 首個請求較慢 | 設定 min_machines_running=1（生產）|
| 資料庫連線中斷 | API 錯誤 | Drizzle 連線池 + 重試 |
